Necessity Review Checklist

Review-only phase
- [ ] Scope is explicit (files + constraints).
- [ ] No code changes were produced.
- [ ] Each candidate has a stable ID (C1..Cn).
- [ ] Each candidate includes location, evidence, and labeled hypotheses.
- [ ] Risk and confidence are stated.
- [ ] Human Decision Gate instructions are provided.

Apply phase (only after approval)
- [ ] Only approved candidates were changed.
- [ ] Change set is minimal and boring.
- [ ] No new abstractions were introduced.
- [ ] Behavior preservation is explicitly considered.
- [ ] Minimal verification commands are provided (or executed if available).
